# Product & UI Principles

> **方針のみ記載**。具体的な画面構成・文言は `docs/ux-flow.md` を参照。

## ターゲット

- **親子（3〜6歳の子供を持つ保護者）**
- 写真を撮って「これなぁに？」→ AI が答える体験

## UX 原則

1. **言葉がなくても伝わる UI**
   - 写真 + アイコン + 状態表示を主体
   - テキストは最小限

2. **モバイル最優先**
   - スマホでの片手操作を前提に設計

3. **CTA は 2ステップ以内**
   - 撮影 → 結果表示まで最短経路を維持
   - 迷わせない導線設計

4. **即時フィードバック**
   - すべての操作に視覚的応答（ローディング、成功、失敗）

## デザイン原則

1. **プレミアム感**
   - 安価に見えないこと
   - 適切な余白、タイポグラフィ、マイクロアニメーション

2. **親向け UI、子向けコンテンツ**
   - 操作するのは親
   - AI の説明文は子どもが理解できる言葉で

## 導線原則

1. **デモ導線は作らない**
   - 登録/ログインに集約
   - 未認証ユーザーは LP → 登録/ログイン

## スコープ制約 (MVP)

- **含まない**: 通知、SNS連携、外部サービス連携
- **含まない**: 画像加工（フィルタ、スタンプ）
- **含む**: 圧縮のみ（品質・サイズ最適化）
