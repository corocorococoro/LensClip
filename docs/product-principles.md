# プロダクト原則

## 提供価値

LensClip は、子どもと過ごす日常の中で生まれる「これなに？」を、親がその場で調べ、あとから親子で見返せる発見の記録へ変える。

価値の中心は画像認識そのものではなく、次の体験が一続きになることにある。

1. 見つける
2. 名前を知る
3. 覚える
4. 集める
5. あとから見返す

## 利用者

- 主な操作者は、小さな子どもを持つ親
- 結果画面は、親が子どもへ読み聞かせながら一緒に見る
- 子どもだけで操作する幼児向けサービスには寄せない

## 体験上の原則

- 撮影から調査開始までの手順を増やさない
- 利用者に見える初期状態を、後続処理で帳尻合わせしない
- 未確定の名前、カテゴリ、設定、権限を近い値へ黙って置換しない
- 処理中、成功、失敗と、次に取れる行動を明確にする
- 写真と発見内容を主役にし、記録が増えるほど図鑑が育つ感覚を伝える
- 親が日常的に使える落ち着きと、親子で見る小さな楽しさを両立する
- 新しいゲーム要素より、既存の記録を見つけやすく、振り返りやすくする

## 視覚言語

- 白または温かいニュートラルを基本のキャンバスにする
- ターコイズは主要操作、選択、進行に一貫して使う
- 補助色は分類、空状態、注意など意味がある場所に限定する
- 装飾、影、枠、文字が写真より目立たないようにする
- 具体的な色値、余白、角丸、コンポーネント表現は `tailwind.config.js`、`resources/css/`、React コンポーネントを一次ソースとする

## 文書化しないもの

実装済み機能の一覧、画面項目、技術スタック、デザイントークンの値はこの文書に複製しない。それらはコードとテストから確認する。
